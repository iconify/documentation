Changing icon palette

This article is part of SVG clean up article.

After initial clean up and validation, the icon palette is changed.

There can only be two types of icons:

  • Icons with a hardcoded palette.
  • Monotone icons, which have only one color.

Icons with palette

Some icons have hardcoded palette:

Colors of these icons cannot be easily changed with a stylesheet.

When such icons are cleaned up, the parser verifies that the icon does not follow system default color (black color for fill) and does not contain currentColor.

Monotone icons

Most icons are monotone. They have only one color, which can be changed.

Move mouse over an example below, it will change color:

When parsing monotone icons, color is changed to currentColor to make sure the icon follows text color. This way it is easy to change color for any icon, regardless if icon uses fill or stroke:

svg {
   color: red;
}

Parsing such icons is not easy. When icons are exported from editors, color is not set to currentColor. Usually monotone icons come with shapes that either use black color or system default color.

Sometimes icons also include white background, which needs to be removed. Parser cleans it up too.

System color

Usually, icons that use fill, do not have any colors:

<svg xmlns="http://www.w3.org/2000/svg" width="24" height="24" viewBox="0 0 24 24">
   <path d="M4 17h2.697L14 21.868V2.132L6.697 7H4c-1.103 0-2 .897-2 2v6c0 1.103.897 2 2 2z"/>
</svg>

Color in those icons is intended to be changed by changing fill property:

svg {
   fill: red;
}

Problem is, many icons use stroke instead of fill, so this approach does not apply to many icon sets, therefore it cannot be used.

Parser modifies those icons by setting fill to currentColor on elements that use system default fill:

<svg xmlns="http://www.w3.org/2000/svg" width="24" height="24" viewBox="0 0 24 24">
   <path d="M4 17h2.697L14 21.868V2.132L6.697 7H4c-1.103 0-2 .897-2 2v6c0 1.103.897 2 2 2z" fill="currentColor"/>
</svg>

Custom color

Usually when you export an icon from editor, it uses whatever color you were using when designing it, usually black:

<svg xmlns="http://www.w3.org/2000/svg" width="24" height="24" viewBox="0 0 24 24">
   <path d="M5 12H19" fill="none" stroke="black" stroke-width="2" stroke-linecap="round"/>
</svg>

Parser changes it to currentColor:

<svg xmlns="http://www.w3.org/2000/svg" width="24" height="24" viewBox="0 0 24 24">
   <path d="M5 12H19" fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round"/>
</svg>

Result

After changes, all shapes that use fill have fill set to currentColor, all shapes that use stroke have stroke set to currentColor, which means icon uses same color as text and can be styled as text:

svg {
   color: red;
}

Mixing palette and currentColor

Icons that mix currentColor and hardcoded palette are not allowed.

Such icons cannot be used in a stylesheet. In stylesheet monotone icons are used as masks, icons with palette are used as background. When the icon component detects currentColor in the icon, it assumes that the icon is monotone and uses it as a mask, displaying it incorrectly.

Also such icons assume that currentColor is either light or dark or cannot be a certain hue. It is just a bad design.

Code

Palette is parsed using the parseColors() function from Iconify Tools.
